<tr><td VALIGN=TOP width="626"><h2 align=center>对Java的新展望</h2>

<h6></h6>
<hr>
	<p>【大纪元3月23日讯】Java自1994年问世以来，每年都有重要进展。今年将是从小到大、开花结果上市的一年，Java手机、Palm/Java PDA、Java Web Start 应用程序、J2EE服务器运算，俨然是一片无所不在的Java王国。时下在微软.net和Sun ONE（开放网络环境）较劲的当口，我们不妨对Java作一下新展望。</p>
<p>对Java的展望，我们就以Java 2的三大部分来说明：</p>
<p>在小版本的J2ME(Java 2 Micro Edition)方面，以前应用的对象是内存容量小、体积也较小的电子装置。小至智能卡、移动电话，都是使用J2ME的最佳平台。在最近的媒体新闻中介绍了Java的八位处理器版本，还有Java在Palm PDA上的应用。相信今年Java在Palm上的应用必是一大卖点，因为将来的PalmOS 4.0将会内含KJava，Sun也会推出针对PalmOS使用的Java版本，所以在市场占有率最大的Palm PDA上开发应用程序，对我们的Java程序员来说是相当方便的。只要用既有的Java程序设计知识，就可以开发出Palm的各类应用系统。Nokia、Motorola、Ericsson以及NTT DoCoMo 都将在今年初推出利用J2ME技术的新手机，所以Java程序员可以大展身手的平台更多了，“Learn Once，Apply Everywhere”这个理想终于可以实现了，技术人员也不用那么辛苦地去学很多不同的东西了。</p>
<p>在中型版本的J2SE(Java 2 Standard Edition)应用上，最新推出的Netscape 6浏览器直接支持Java 2环境。对市场使用率更大的IE用户来说，除了原来Sun提供的Java Plug-in外，Sun现在又推出了另外一个新的解决方案，称为Java Web Start。原来的Java Applet是在WebBrowser 中开出一块方形区域来执行Java程序，但这样在执行效能和相容性上都受限于原有的Web Browser。现在新推出的Java Web Start则是在操作系统上直接执行的Java Application，当然也可在网页上启动。这样既能与网页结合，在执行上也更快、更有效率。Java Web Start的推出可以消除一般人对既有Java Applet执行上的疑虑，相信对一般电脑上执行的客户端Java应用系统的开发是一大利器。所以在今年，Java Web Start是一大重点。</p>
<p>另外，在大型的J2EE(Java 2 Enterprise Edition)的应用上，可以说“J2EE”已经成为服务器运算环境的标准。Java Servlets、JSP(Java ServerPages)、EJB(Enterprise JavaBeans)、JavaMail、JDBC等等，都是各厂商产品开发的重点方向。所以市场上未来的服务器运算的解决方案只会有两个选择：J2EE或将来微软的.NET。J2EE相容的是一般Intel个人电脑(Linux、Windows&#8230;..)、Mac以及各家高效能高稳定度的UNIX服务主机，而.NET只是应用于微软的Win32平台。今后客户的选择方向相信已经十分清楚。除了这三大Java组合外，Java和XML的整合也是未来的重点。Sun已经公布了几套Java的XML API规格，将会合并在下一版本的标准JDK中。所以在Java程序中可以完全处理XML文件。Java这个跨平台的开发环境，加上XML这个跨平台的数据格式，如虎添翼地结合成了一个最佳的跨平台解决方案。</p>
